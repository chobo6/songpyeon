# 임시 닉네임 + 하단 팀 패널 설계 (2026-07-15)

## 배경

지금은 로그인/계정 시스템이 전혀 없고, Colyseus가 접속마다 발급하는 `sessionId`로만
플레이어를 구분한다. 실제 플레이 화면에서도 "team-1 팀의 차례입니다" 처럼 팀 ID로만
표시되어 같이 하는 사람이 누구인지 구분이 안 된다.

이번 작업은 계정/인증 시스템이 아니라, **그때그때 입력하는 임시 닉네임**으로 화면에서
사람을 구분할 수 있게 하는 것이 목적이다. 서버에 영구 저장하지 않고, 같은 브라우저 탭이
유지되는 동안만 기억한다.

## 요구사항

1. "온라인" 선택 직후, 서버 접속 전에 닉네임 입력 화면을 보여준다 (필수 입력).
2. 닉네임은 최대 10자, 앞뒤 공백 제거. 같은 탭에서 다시 "온라인"을 선택하면 이전에
   입력한 닉네임이 입력창에 기본값으로 채워진다 (수정만 하면 됨) — `sessionStorage`에
   저장, 탭을 닫으면 사라짐.
3. 대기실(RoleSelect)에서 이미 역할을 고른 플레이어의 자리에 닉네임을 보여준다.
4. 플레이 화면 하단 영역(현재 `ButtonPanel`이 있는 자리)의 표시 내용을 상황에 따라 바꾼다:
   - **내 팀 차례(내가 눌러야 할 때)**: 지금처럼 색상 버튼판을 보여준다.
   - **다른 팀/팀원 차례(관전 중), 또는 탈락 후 관전**: 버튼판 대신, 두 팀의 돼지/토끼
     자리별로 캐릭터 아이콘 + 닉네임 + 절구(생명) 개수를 보여주는 패널을 띄운다
     (참고 스크린샷의 카드형 레이아웃 — 단, 채팅/이모지/전송 버튼, 카드 테두리 색상,
     닉네임 글자색 등 세부 스타일은 이번 범위에서 제외하고 기본 돼지/토끼 이미지만 사용).
5. 채팅 기능은 이번 범위에서 제외 (다음에 별도로).

## 아키텍처

### 서버

`PlayerState`에 `nickname: string` 필드 추가. `onJoin(client, options)`에서
`options.nickname`을 받아 다듬어서(trim, 10자 제한, 빈 값/공백뿐이면 기본값 "플레이어")
저장한다 — 클라이언트가 보낸 값은 신뢰 경계를 넘어오는 입력이므로 서버에서도 검증/제한
한다 (클라이언트의 10자 제한은 UX용, 서버 제한이 실제 방어선).

### 클라이언트 접속 흐름

지금 `App.tsx`의 `OnlineFlow`는 마운트되자마자 `useMatchRoom()`을 호출해 바로 서버에
접속한다. 여기에 닉네임 입력 단계를 끼워넣는다:

```
ModeSelect ("온라인" 클릭)
  → OnlineFlow
      ├─ 닉네임 없으면: NicknameEntry 화면 (sessionStorage에 저장된 이전 값 있으면 기본값)
      └─ 닉네임 확정되면: 기존 흐름(useMatchRoom → joinMatch(nickname) → RoleSelect → Game)
```

`client/src/colyseus.ts`의 `connectToMatch()`가 닉네임을 받아
`client.joinOrCreate("match", { nickname })`로 전달한다 (재접속 경로는 기존 PlayerState를
그대로 복구하므로 닉네임을 다시 보낼 필요 없음).

### RoleSelect

`room.state.players`에서 이미 다른 플레이어들의 `nickname`도 동기화되어 오므로, 각 팀의
채워진 돼지/토끼 자리에 그 플레이어의 닉네임을 표시한다 (지금은 숫자 카운트만 표시).

### 플레이 화면 하단 패널 전환

`Game.tsx`가 지금 "내 턴이면 MyTurnScreen, 아니면 SpectatorScreen"으로 분기하는데,
공통으로 쓰이던 `TeamStatusBar`(절구 표시, 보드 위쪽)를 없애고, 대신 하단 영역에 상황별로
다른 컴포넌트를 넣는다:

- `MyTurnScreen`: 하단에 `ButtonPanel` (기존과 동일)
- `SpectatorScreen`(관전 중이든 탈락 후든): 하단에 새 `TeamRosterPanel` — 팀별로 열을
  나눠 각 열에 돼지 카드(역할 기본 이미지 + 닉네임) / 토끼 카드를 위아래로 배치하고,
  그 팀 열 안에 절구 개수를 한 번만 표시한다 (카드마다 중복 표시하지 않음 — 절구는
  플레이어가 아니라 팀 단위 자원이므로).

`TeamRosterPanel`은 `ButtonPanel`의 `panelBg`(하단 나무틀 배경)를 그대로 재사용해서
두 화면이 전환될 때 배경이 튀지 않게 한다.

## 명시적 범위 제외

- 채팅 (다음 작업)
- 카드 테두리 색, 닉네임 글자색 등 세부 스타일 (참고 스크린샷의 대략적인 배치만 따라감)
- 닉네임 서버 영구 저장, 계정/인증, 중복 닉네임 방지
- 부적절 단어 필터링

## 테스트 계획

- **서버**: `onJoin`에 닉네임 옵션을 넣었을 때 trim/길이 제한/기본값 처리가 되는지
  `MatchRoom.test.ts`에 케이스 추가.
- **클라이언트**: 자동화 테스트 없음(기존 관례) — Playwright로 닉네임 입력 → 대기실에
  닉네임 표시 → 플레이 중 하단 패널이 내 턴/남의 턴에 따라 버튼판⇄로스터로 전환되는지
  수동 확인.
