# 대기실/관전 채팅 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 스펙 문서(`docs/superpowers/specs/2026-07-15-chat-design.md`) 참고. 대기실(RoleSelect)과
관전 화면(SpectatorScreen)에 독립된 채팅을 추가한다.

**Architecture:** 서버 `MatchState`에 `lobbyChat`/`matchChat` 두 개의 독립 메시지 배열(각 최근 50개
캡). 클라이언트는 화면 무관 재사용 `ChatBox` 컴포넌트 하나로 두 화면에 렌더링.

**Tech Stack:** 기존 스택 그대로 (Colyseus Schema, React).

## Global Constraints

- 채팅은 대기실/관전 화면에만 노출한다 — 내 차례 화면(`MyTurnScreen`)에는 절대 안 보이게 한다
  (읽기 전용 포함 금지).
- 로비 채팅과 관전 채팅은 서로 완전히 독립된 기록이다 — 섞이지 않는다.
- 닉네임은 클라이언트가 채팅 전송 시 같이 안 보낸다 — 서버가 `this.state.players.get(client.sessionId)`로
  이미 알고 있는 닉네임을 그대로 쓴다 (클라이언트가 다른 닉네임을 사칭해서 보낼 수 없게).
- 이모지 피커, "+" 첨부 버튼, 모더레이션(욕설 필터 등)은 이번 스코프에 없다.

const MAX_CHAT_LENGTH = 100;

// Unlike sanitizeNickname (which always returns something displayable),
// an empty/invalid chat message has nothing sensible to fall back to — the
// caller must simply drop it, so this returns null instead of a placeholder.
export function sanitizeChatText(input: unknown): string | null {
  if (typeof input !== "string") return null;
  const trimmed = input.trim().slice(0, MAX_CHAT_LENGTH);
  return trimmed || null;
}

새 private 메서드 두 개 추가 (`handleChooseRole` 근처, 아무 위치나 클래스 안이면 됨):
```ts
private handleSendChat(client: Client, rawText: unknown) {
  const text = sanitizeChatText(rawText);
  if (!text) return;

  const player = this.state.players.get(client.sessionId);
  if (!player) return;

  const list = this.state.phase === "lobby" ? this.state.lobbyChat : this.state.matchChat;
  this.pushChat(list, player.nickname, text);
}

private pushChat(list: ArraySchema<ChatMessage>, nickname: string, text: string) {
  const message = new ChatMessage();
  message.nickname = nickname;
  message.text = text;
  message.sentAt = Date.now();
  list.push(message);
  if (list.length > MAX_CHAT_MESSAGES) list.shift();
}
```

안 들어와 있을 수 있음 — `pushChat`의 매개변수 타입 주석에 필요하므로 import 추가:

- [ ] **Step 2: `client/src/components/ChatBox.tsx` 작성**

```tsx
import { useEffect, useRef, useState } from "react";
import type { ChatMessage } from "../game/matchTypes";
import styles from "./ChatBox.module.css";

export function ChatBox({
  messages,
  onSend,
}: {
  messages: ChatMessage[];
  onSend: (text: string) => void;
}) {
  const [draft, setDraft] = useState("");
  const listRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const el = listRef.current;
    if (el) el.scrollTop = el.scrollHeight;
  }, [messages.length]);

  function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    const trimmed = draft.trim();
    if (!trimmed) return;
    onSend(trimmed);
    setDraft("");
  }

  return (
    <div className={styles.wrap}>
      <div className={styles.list} ref={listRef}>
        {messages.length === 0 && <p className={styles.empty}>아직 채팅이 없어요</p>}
        {messages.map((m, i) => (
          <p key={i} className={styles.line}>
            <span className={styles.nickname}>{m.nickname}</span>
            <span className={styles.text}>{m.text}</span>
          </p>
        ))}
      </div>
      <form className={styles.inputRow} onSubmit={handleSubmit}>
        <input
          className={styles.input}
          value={draft}
          onChange={(e) => setDraft(e.target.value)}
          maxLength={100}
          placeholder="메시지 입력"
        />
        <button className={styles.sendButton} type="submit" disabled={!draft.trim()}>
          전송
        </button>
      </form>
    </div>
  );
}
```

      </div>

      </button>
    </div>
  );
}
