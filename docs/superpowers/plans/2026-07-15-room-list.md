# 방 목록(공개방) + 로비 재접속 유예 Implementation Plan

> **For agentic workers:** 시간 압박(실사용자 대기 중)으로 컨트롤러가 직접 구현. subagent-driven-development 미적용 — 단일 태스크로 밀접하게 결합된 변경이라 분리 이득이 없음.

**Goal:** 스펙 문서(`docs/superpowers/specs/2026-07-15-room-list-design.md`) 참고. quick-join을 방 목록 화면으로 대체하고, 로비 단계 재접속 유예를 추가한다.

## Global Constraints

- 이 Colyseus 버전(0.16.x)의 `colyseus.js`에는 `client.getAvailableRooms()`가 없다 — 서버에
  `GET /api/rooms` 커스텀 라우트를 추가하고 `matchMaker.query({ name: "match" })`(서버 내부
  matchmaker API, `IRoomCache[]` 반환: `roomId`/`clients`/`maxClients`/`locked`/`metadata`)를
  호출해서 구현한다.
- 이 라우트는 `server/src/createServer.ts`의 SPA catch-all(`app.get(/.*/, ...)`)보다 **먼저**
  등록해야 한다 (안 그러면 catch-all이 먼저 매치해서 `index.html`을 반환해버림).
- 클라이언트가 `/api/rooms`를 호출할 땐 프로덕션(서버가 정적파일도 같이 서빙)에서는 상대경로로
  같은 origin을 치면 되지만, 로컬 dev(`vite`가 5173, server가 2567로 분리)에서는 서버 주소를
  명시해야 한다 — 기존 `client/src/colyseus.ts`의 `endpoint`(ws://.../wss://...) 값에서 프로토콜만
  `ws`→`http`로 바꿔 재사용한다 (같은 호스트:포트를 가리키므로 새로 계산할 필요 없음).

**Files:**

**단계:**

1. `MatchRoom.ts`의 `MatchRoomOptions`에 `nickname?: unknown` 추가, `onCreate`에 메타데이터 설정
   추가:
```ts
interface MatchRoomOptions {
  turnDurationMs?: number;
  nickname?: unknown;
}
```
```ts
onCreate(options: MatchRoomOptions = {}) {
    if (options.turnDurationMs) this.turnDurationMs = options.turnDurationMs;
    this.setMetadata({ hostNickname: sanitizeNickname(options.nickname) });
    // ...(이하 기존 코드 그대로)
```

2. `onLeave`에서 `phase === "playing"` 조건 제거 (로비 단계에도 재접속 유예 적용):
```ts
async onLeave(client: Client, consented: boolean) {
    if (!consented) {
      try {
        await this.allowReconnection(client, RECONNECTION_GRACE_SECONDS);
        return;
      } catch {
        // grace period expired without a reconnect — fall through to removal.
      }
    }

    this.removePlayer(client.sessionId);
}
```

3. `createServer.ts`에 `GET /api/rooms` 라우트 추가 (catch-all보다 먼저):
```ts
import express from "express";
import { createServer as createHttpServer } from "http";
import path from "path";
import { fileURLToPath } from "url";
import { Server, matchMaker } from "colyseus";
import { WebSocketTransport } from "@colyseus/ws-transport";
import { MatchRoom } from "./rooms/MatchRoom";

const __dirname = path.dirname(fileURLToPath(import.meta.url));
const clientDistPath = path.join(__dirname, "../public");

export function createGameServer(): Server {
  const app = express();
  app.use(express.static(clientDistPath));

  app.get("/api/rooms", async (_req, res) => {
    const rooms = await matchMaker.query({ name: "match" });
    res.json(
      rooms.map((r) => ({
        roomId: r.roomId,
        clients: r.clients,
        maxClients: r.maxClients,
        locked: r.locked,
        hostNickname: (r.metadata as { hostNickname?: string } | undefined)?.hostNickname ?? "?",
      })),
    );
  });

  const httpServer = createHttpServer(app);
  const gameServer = new Server({
    transport: new WebSocketTransport({ server: httpServer }),
  });
  gameServer.define("match", MatchRoom);

  app.get(/.*/, (_req, res) => {
    res.sendFile(path.join(clientDistPath, "index.html"));
  });

  return gameServer;
}
```

**검증:** `cd server && npm test` (기존 `MatchRoom.test.ts`가 여전히 통과해야 함 — `onCreate`/`onLeave`
변경이 기존 동작을 깨지 않는지 확인). 서버 기동 후 `curl localhost:2567/api/rooms` → 빈 배열 `[]`
확인.

### Task 2: 클라이언트 — 방 목록 화면 + 생성/입장 흐름

**Files:**
- Modify: `client/src/colyseus.ts`
- Modify: `client/src/game/useMatchRoom.ts`
- Modify: `client/src/App.tsx`
- Modify: `client/src/components/Game.tsx` (변경 없을 수도 있음 — 확인 후 필요시만)
- Create: `client/src/components/RoomList.tsx`
- Create: `client/src/components/RoomList.module.css`

**요구사항:**
- `colyseus.ts`: `joinOrCreate` 기반 `connectToMatch`를 `create`/`joinById` 두 경로로 대체하는
  join spec 타입 도입. `listRooms()` 함수 추가 (`fetch`로 `/api/rooms` 호출, 로컬 dev에서는
  `endpoint`의 `ws`→`http` 치환한 절대경로 사용).
- `useMatchRoom.ts`: `nickname: string` 파라미터를 join spec으로 교체. `leaveAndRejoin`/`generation`은
  더 이상 필요 없음(방 목록 화면으로 돌아가는 게 새 "재접속" 개념이 됨) — 제거하고 이펙트를 마운트 시
  1회만 실행하도록 단순화. 기존 stale 주석("no reconnection grace during lobby")도 Task 1에서
  사실이 아니게 됐으므로 갱신.
- `RoomList.tsx`: 2초 간격 폴링, 카드에 `"{hostNickname}의 방 (clients/maxClients)"`, `locked`면
  "게임 중"+비활성화, `!locked && clients >= maxClients`면 "가득 참"+비활성화, 그 외엔 "입장" 활성화.
  상단에 "새 방 만들기" 버튼.
- `App.tsx`: `OnlineFlow`에 닉네임 다음 단계로 `RoomList` 삽입, 방 생성/선택 시 `ConnectedOnlineFlow`로
  전환. `ConnectedOnlineFlow`의 나가기(`onExit`)는 이제 완전히 앱을 벗어나지 않고 `RoomList`로
  돌아가도록 재구성. `Game`에 전달되는 `onLeave`/`onExit` 둘 다 이 "방 목록으로 돌아가기" 핸들러를
  쓰도록 통일.

**검증:** `npm run build --workspace client` 타입에러 없이 통과. `npm run lint --workspace client`
통과. 로컬 `docker build`+`docker run`으로 실제 브라우저 검증 — 방 만들기, 다른 탭에서 목록에 뜨는지,
입장, 나가기 후 목록으로 돌아오는지, 로비에서 네트워크를 잠깐 끊었다 붙여도(DevTools offline
토글) 다른 탭에서 계속 보이는지까지 확인.
