# BGM 재생 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 온라인 매치 진행 중(`room.state.phase === "playing"`)에만 배경음악이 끊김 없이 반복 재생되게 한다.

**Architecture:** `client/public/game-assets/audio/bgm.mp3`(이미 추출되어 배치됨, git 제외 대상)를 재생하는 `<audio>` 래퍼 컴포넌트 `BgmPlayer`를 새로 만들고, `Game.tsx`에서 `phase === "playing"`일 때 `MyTurnScreen`/`SpectatorScreen`과 같은 위치의 형제 요소로 렌더링한다. 두 화면 컴포넌트는 턴마다 서로 전환되지만 `BgmPlayer`는 JSX 트리 상 항상 같은 자리에 있으므로 React가 리마운트하지 않는다 — 이 성질이 "턴 전환 시 BGM 재시작 안 됨" 요구사항을 그대로 만족시킨다.

**Tech Stack:** React 19 (client), 순수 HTML `<audio>` 엘리먼트 (별도 라이브러리 불필요).

## Global Constraints

- 재생 시점은 `room.state.phase === "playing"`일 때만 — 로비/대기실/방 목록 화면에서는 재생하지 않는다 (`docs/superpowers/specs/2026-07-16-bgm-design.md`).
- 턴 전환(`MyTurnScreen` ↔ `SpectatorScreen`)마다 BGM이 재시작되면 안 된다.
- 끝까지 재생되면 처음부터 자동 반복(loop)된다.
- 볼륨은 기본 40%(`0.4`), 별도 조절 UI는 이번 스코프에 없다.
- 음소거/볼륨 컨트롤 UI는 이번 스코프에 없다.
- 자동재생 관련 별도 처리(음소거 시작 후 unmute 등)는 불필요 — 게임 시작 시점엔 이미 여러 차례 클릭을 거친 뒤라 브라우저의 사용자 상호작용 요구 정책에 문제없다.
- `client/public/game-assets/audio/bgm.mp3`는 저작권 있는 리소스라 `client/.gitignore`의 `public/game-assets/` 규칙으로 이미 git 제외 대상이다 — 이 경로를 참조하는 코드만 커밋하면 되고, 파일 자체를 add하려 하면 안 된다.

**Files:**

**Interfaces:**
- Consumes: 없음 (props 없는 컴포넌트)
- Produces: `BgmPlayer` — named export, `() => JSX.Element`. Task 2가 이 컴포넌트를 import해서 렌더링한다.

- [ ] **Step 1: `BgmPlayer.tsx` 작성**

```tsx
import { useEffect, useRef } from "react";

const BGM_VOLUME = 0.4;

export function BgmPlayer() {
  const audioRef = useRef<HTMLAudioElement>(null);

  useEffect(() => {
    const audio = audioRef.current;
    if (audio) audio.volume = BGM_VOLUME;
  }, []);

  return (
    <audio ref={audioRef} src="/game-assets/audio/bgm.mp3" loop autoPlay />
  );
}
```

`volume`은 HTML 표준 속성이 아니라 DOM 프로퍼티라서 JSX 속성으로 줄 수 없다 — `ref` + `useEffect`로 마운트 시 한 번 설정한다.

### Task 2: `Game.tsx`에 통합하고 끊김 없음을 검증

**Files:**
- Modify: `client/src/components/Game.tsx`

**Interfaces:**
- Consumes: `BgmPlayer` (Task 1의 `client/src/components/BgmPlayer.tsx`에서 import)

- [ ] **Step 1: `Game.tsx`를 읽고 현재 구조 확인**

현재 `Game.tsx`는 `phase === "lobby"`면 즉시 `<RoleSelect>`를 반환하고, 그 외(`"playing"`)에는 `isMyTeamActive` 여부에 따라 `<MyTurnScreen>` 또는 `<SpectatorScreen>`을 반환한다 (각각 `if`문의 개별 `return`). `BgmPlayer`를 이 두 반환문과 같은 자리의 형제로 만들려면, 두 분기를 하나의 지역 변수로 모으고 `<BgmPlayer>`와 함께 프래그먼트로 묶어 반환하도록 바꿔야 한다.

- [ ] **Step 2: `Game.tsx` 수정**

`client/src/components/Game.tsx` 전체를 다음으로 교체:

```tsx
import type { Room } from "colyseus.js";
import type { MatchState } from "../game/matchTypes";
import { RoleSelect } from "./RoleSelect";
import { MyTurnScreen } from "./MyTurnScreen";
import { SpectatorScreen } from "./SpectatorScreen";
import { BgmPlayer } from "./BgmPlayer";

export function Game({
  room,
  onLeave,
  onExit,
}: {
  room: Room<MatchState>;
  onLeave: () => void;
  onExit: () => void;
}) {
  const { phase } = room.state;

  if (phase === "lobby") return <RoleSelect room={room} onExit={onExit} />;

  const me = room.state.players.get(room.sessionId);
  const activeTeam = room.state.teams[room.state.activeTeamIndex];
  // activeTeam can itself be eliminated once every team has been wiped out
  // (the server freezes turns at that point instead of ending the match) —
  // that team's own players fall through to SpectatorScreen too, since
  // there's no turn left for anyone to take.
  const isMyTeamActive = me?.teamId === activeTeam?.id && !activeTeam?.eliminated;

  let screen = null;
  if (me && activeTeam && isMyTeamActive) {
    screen = <MyTurnScreen room={room} me={me} />;
  } else if (activeTeam) {
    const myTeam = room.state.teams.find((t) => t.id === me?.teamId);
    screen = (
      <SpectatorScreen
        room={room}
        activeTeam={activeTeam}
        eliminated={myTeam?.eliminated ?? false}
        onLeave={onLeave}
      />
    );
  }

  // BgmPlayer stays at this fixed position in the tree across every
  // MyTurnScreen <-> SpectatorScreen switch (every turn), so React never
  // remounts it while phase stays "playing" — that's what keeps the BGM
  // from restarting each turn.
  return (
    <>
      <BgmPlayer />
      {screen}
    </>
  );
}
```

- [ ] **Step 4: 로컬 dev 서버로 수동 확인 (서버+클라이언트 동시 실행 필요)**

Run: `npm run dev` (루트에서, server 2567 + client 5173 동시 기동)

브라우저(또는 Playwright)로 `http://localhost:5173` 접속 후:
1. 온라인 모드 진입 → 닉네임 입력 → 방 만들기 → 역할 선택까지는 소리가 안 나야 함 (phase가 아직 `"lobby"`).
2. 4명(4개 탭)이 모두 역할을 고르면 `phase`가 `"playing"`으로 바뀌면서 BGM이 재생 시작해야 함.
3. 턴이 넘어가서 내 화면이 `MyTurnScreen` ↔ `SpectatorScreen`으로 전환될 때 BGM이 끊기거나 처음으로 되감기지 않고 계속 이어져야 함 — 이게 이번 작업에서 가장 중요하게 검증해야 하는 부분이다.

이 3번 항목은 육안/청취만으로는 놓치기 쉬우므로, Playwright의 `browser_evaluate`로 턴 전환 전후 `document.querySelector("audio").currentTime` 값을 각각 읽어 두 번째 값이 첫 번째 값보다 크면서(재생이 계속됨) `0` 근처로 리셋되지 않았는지(리마운트되지 않았음) 확인한다.
