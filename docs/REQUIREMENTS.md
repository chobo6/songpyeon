## 0. 배경

- 카카오톡 오픈채팅봇과 **무관**. **"마피아42"**라는 게임의 이벤트 미니게임 **"송편만들기대작전"**을 웹으로 이식하는 프로젝트.
- 참고 영상(https://www.youtube.com/watch?v=NFTJ8jenKTk)은 채팅 오버레이가 실제 게임 화면을 가리고 있어 핵심 규칙 파악에 쓸 수 없었음 → **핵심 규칙은 사용자의 직접 설명이 1차 소스**.

## 1. 게임 구조 개요

- 팀전. **팀당 인원은 2명(돼지 1 + 토끼 1) 고정**, **팀 개수는 방 생성 시 1~3팀 중 선택**(총 2~6명) — §10-2에서 "추후 확장"으로 열어뒀던 항목, 구현 완료(`server/src/game/teamCount.ts`, 방 목록 화면의 1팀/2팀/3팀 버튼).
- 각 플레이어는 **게임 시작 전** 역할(돼지/토끼)을 직접 선택.
- **팀 단위로 턴을 진행**한다 (개인 턴이 아님). 라운드는 모든 팀이 한 번씩 턴을 마치면 1 증가.
- 팀이 절구 5개(생명, **팀 공유 자원**)를 모두 잃으면 해당 팀 탈락.
- 탈락한 팀은 라운드 순환에서 건너뛰어지며, 관전하거나 "나가기"로 매치를 떠나 새 큐에 재진입할 수 있음.
- **한 팀만 남아도 게임은 종료되지 않고 생존 팀이 계속 라운드를 이어감** (v0.6). "승리"라는 개념 자체가 없음 — 상대 탈락은 조용히 처리되고 게임은 계속됨. 두 팀 모두 탈락하면(동시 탈락) 그 시점에서 멈춤.
- 코인/화폐 시스템 없음. 진행바·보상 팝업·수집 보드 등 메타 시스템도 **범위 제외** (§9).

## 2. 역할

| 역할 | 담당 색상 (4가지) |
|---|---|
| 돼지 | 빨강, 주황, 노랑, 보라 |
| 토끼 | 민트, 초록, 파랑, 분홍 |

- 역할 배정: 게임 시작 전 플레이어가 직접 선택.

## 3. 팀 턴 진행 방식 (확정)

- 팀의 턴이 되면 **4초의 제한시간**이 주어진다. (개인이 아니라 **팀 전체**에 4초.)
- 팀원 두 명은 **같은 화면(하나로 합쳐진 버튼 시퀀스)**을 함께 보고, 시퀀스는 **좌→우, 위→아래의 절대 순서**로 하나씩 "다음 차례" 버튼이 정해진다.
- 이 절대 순서는 **색상과 무관하게 시퀀스 전체를 관통**한다. 즉 지금 차례인 버튼이 상대 팀원의 색상이면, 해당 팀원이 그 버튼을 누를 때까지 다음 버튼(내 색상이라도)은 아직 "차례"가 아니다.
- 두 팀원은 각자 **자신의 역할 색상 버튼이 차례가 될 때마다** 즉시 눌러야 한다 — 병렬로 나눠서 동시에 처리하는 게 아니라, **정해진 순서를 따라 담당자가 번갈아 개입하는 협동 릴레이** 구조. 실시간으로 시퀀스를 계속 주시하고 있다가 자기 차례가 오면 놓치지 않고 눌러야 하므로 "실시간성"이 중요함.
- 예: 시퀀스가 `[노랑,보라, 초록,파랑, 빨강,보라, 민트,민트, ...]` 라면 → 노랑(돼지)→보라(돼지)→초록(토끼)→파랑(토끼)→빨강(돼지)→보라(돼지)→민트(토끼)→민트(토끼) 순서로 **한 번에 한 버튼씩만** 유효하며, 두 팀원이 색상에 따라 번갈아 누른다.
- 라운드가 올라가며 시퀀스가 18개, 24개, 30개로 늘어나도 4초 안에 처리 가능하다고 확인됨 — 두 팀원이 시퀀스를 밀착 주시하며 빠르게 번갈아 누르는 것이 전제.

## 4. 중앙 버튼 시퀀스 생성 규칙

팀 턴의 시퀀스는 아래 돼지/토끼 규칙에 따라 생성된 조각들이 섞여 하나의 시퀀스를 이룬다. 각 팀원은 이 중 자기 역할 색상에 해당하는 조각만 처리한다.

### 4.1 돼지 색상 조각

- 빨강/주황/노랑 버튼 하나하나마다 바로 뒤에 보라가 붙는 **`[색상, 보라]` 쌍** 단위로 구성.
  - 예: `[노랑,보라]`, `[빨강,보라]`, `[주황,보라]`
- 어떤 색이 나올지, 쌍들의 등장 순서는 임의(랜덤).

### 4.2 토끼 색상 조각

- **민트**: 동일 색이 연속으로 **2 / 4 / 6개** 단위로 등장.
- **초록 / 파랑 / 분홍**: 임의 순서로 등장하되, 항상 **2개가 한 쌍**으로 붙어서 나옴 (같은 색 반복도, 다른 색 조합도 가능). 예: `[초록,파랑]`, `[분홍,파랑]`, `[초록,초록]`

## 5. 시퀀스 길이 성장 규칙

- **1라운드는 3줄(6버튼 × 3줄 = 18버튼)로 시작**
- **10라운드마다 1줄(6버튼)씩 증가**
- 4초 제한과의 정합성은 §3의 2인 협동 릴레이 구조로 해소됨 (해결됨 — 더 이상 열린 질문 아님).

## 6. 관전 패널

- 우측 상단 관전 패널은 **현재 턴인 팀의 진행 상황을 실시간으로 보여주는 미러(mirror)**.
- 한 줄당 버튼 6개, §5의 성장 규칙과 동일하게 라운드에 따라 줄 수 증가.

## 7. 생명(절구) 시스템

- **팀당 5개, 팀 공유 자원** (확정).
- 시간초과 또는 오답 버튼 입력 시 1개씩 소멸.
- 0개가 되면 팀 탈락.

## 8. 성공/실패 피드백

- 시퀀스를 시간 안에 다 처리하면 **"성공!" 표시만 뜨고 별다른 보상은 없음** (점수·재화 등 추가 보상 없음).
- 실패(시간초과 또는 오답)하면 절구가 1개 소멸.

## 12. 화면 와이어프레임 (영상 기반 — "내 차례" 화면)

참고 영상 02:56~03:02 구간(ROUND 78)이 실제로 팀 턴 중 플레이어에게 보이는 화면임을 확인함. 채팅 로그 화면과 레이아웃 뼈대(상단 헤더)는 공유하되, 팀 턴 중에는 중앙~하단이 아래처럼 바뀐다.

### 12.1 레이아웃 구성

1. **상단 헤더** (채팅 화면과 공통): 마스코트 아이콘 + 좌상단 숫자(코인 아님, **누적 성공 횟수**) + ROUND 카운터 + 진행바.
2. ~~플레이버 타이틀("OOO송편")~~ → **범위 제외** (원본 게임 전용 아이템, §9).
3. **메인 시퀀스 보드**: 원형 토큰이 한 줄에 6개씩, 여러 줄(§5 성장 규칙과 동일하게 라운드에 따라 줄 수 증가)로 배치되고 같은 줄 안에서 선으로 연결됨. 진행됨에 따라 처리된 토큰부터 순서대로 사라짐. 커서 아이콘이 현재 처리 대상 위치를 표시.
4. **하단 버튼 패널** (자신의 역할에 맞게 렌더링됨): 총 6개 슬롯. ~~2행 3열 그리드~~ → **정정 (v0.6)**: 실제 화면 배치는 왼쪽 삼각형(▲)과 오른쪽 역삼각형(▽) 두 그룹 — 자세한 내용은 §12.2 참고. 자기 역할 색상 4개는 채워진 버튼으로, 나머지 2개 슬롯은 빈 원으로 표시.
5. **턴 종료 시**: 시퀀스를 다 처리하면 "성공!" 표시 (§8 피드백과 동일 이벤트, 플레이버 문구 없이 단순 표시).

### 12.2 버튼 패널 슬롯 ↔ 색상 매핑 (확정)

6개 슬롯(2행 3열) 기준. 같은 물리적 슬롯이 역할에 따라 다른 색으로 렌더링되거나, 한쪽 역할에서만 쓰이고 다른 쪽에선 빈 원으로 남는다.

| 슬롯 위치 | 돼지 화면 | 토끼 화면 |
|---|---|---|
| 상단-좌 | 빨강 | (빈 원) |
| 상단-중 | (빈 원) | 초록 |
| 상단-우 | (빈 원) | 파랑 |
| 하단-좌 | 노랑 | 민트 |
| 하단-중 | 주황 | (빈 원) |
| 하단-우 | 보라 | 분홍 |

- 노랑↔민트, 보라↔분홍은 같은 슬롯을 공유(역할에 따라 다른 색이 그려짐).
- 빨강·주황은 돼지 전용 슬롯(토끼 화면에서는 빈 원), 초록·파랑은 토끼 전용 슬롯(돼지 화면에서는 빈 원).
- **화면상 배치 (v0.6 갱신)**: 위 표의 슬롯 이름(상단-좌 등)은 그대로 유지하되, 실제 화면은 균등한 2행3열 그리드가 아니라 **왼쪽 삼각형(▲: 상단-좌가 꼭짓점, 하단-좌/하단-중이 밑변) + 오른쪽 역삼각형(▽: 상단-중/상단-우가 윗변, 하단-우가 꼭짓점)** 두 그룹으로 배치됨. 돼지는 왼쪽 삼각형이 꽉 차고(빨강 꼭짓점 + 노랑·주황 밑변) 오른쪽 역삼각형은 꼭짓점(보라)만 채워짐. 토끼는 오른쪽 역삼각형이 꽉 차고(초록·파랑 윗변 + 분홍 꼭짓점) 왼쪽 삼각형은 밑변 좌측(민트)만 채워짐.

### 12.3 실제 관찰된 시퀀스 예시 (ROUND 78, "토끼송편")

- 1행: 민트 × 6 (토끼 담당, §4.2의 6개 단위 민트 런)
- 2행: 노랑,보라,노랑,보라,주황,보라 (돼지 담당, §4.1의 `[색상,보라]` 쌍 3개)
- 3~4행: 이어지는 돼지 조각 (마지막까지 관측되지 않았으나 동일 패턴으로 추정)
- 처리 순서(§3의 절대 순서 규칙과 일치): 토끼 플레이어가 먼저 민트 6개를 전부 처리 → 그 다음 돼지 플레이어가 노랑→보라→노랑→보라→주황→보라 순으로 처리.
- 이 예시에서는 공교롭게도 "줄 경계"와 "역할 조각 경계"가 일치했지만(1행=순수 민트, 2행=순수 돼지), 항상 그런지는 불명 — 화면 표시상 줄바꿈은 6개 단위로 기계적으로 이루어지고, 역할 조각은 그와 무관하게 이어붙는 구조로 구현하는 것이 안전.
